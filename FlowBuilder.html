How data works

Your flow has two layers:

  • The design — the boxes, arrows, names and value ratings. You build this once. → Share with team hands out a read-only copy; Back up design saves the blueprint as JSON (backup / move to another machine — not the data step).
  • The numbers — wait time, touch time, WIP per step. These change constantly and come from a spreadsheet. → Data template → Connect → Refresh.

The data loop — how to keep it live:

  1. Build your flow. Each step has a StepID (its join key — editable in the inspector as "Data key").
  2. ⬇ Data template — downloads a CSV with the exact columns this flow needs and one row per step (its StepID + name). This file is the spec.
  3. A teammate/programmer fills that template (or points a query/export at the same columns + StepIDs) and keeps it updated in a OneDrive/SharePoint-synced folder.
  4. 🔌 Connect data to that file once — it's remembered. Then ⟳ Refresh (or just reopen the flow) pulls the latest. The status shows “N/N matched” so you know the file lines up.

Template / spreadsheet columns (one row per step × segment × period):
StepID · StepName · NodeType · Segment · Period · TouchTimeAvg · WaitTimeAvg · WIP · Volume

If your browser can't remember the file (some setups), Refresh simply re-opens the picker — still one click.
